## Архитектура и основные компоненты

### Основная структура приложения

FatSecret построен на модульной архитектуре с четырьмя главными вкладками:

**1. Вкладка "Дневник"**
- Центральный компонент для ввода пищевых данных
- Отображение приемов пищи (завтрак, обед, ужин, перекусы)
- Реал-тайм расчет потребленных калорий и оставшейся нормы
- Интеграция с системой упражнений и активности
- Отображение соотношения БЖУ в текущий день

**2. Вкладка "Я" (Профиль)**
- Управление профилем пользователя
- Отслеживание веса (график "Мой вес")
- Управление напоминаниями о взвешивании и приемах пищи
- Фотоальбом продуктов с распознаванием изображений
- Настройки конфиденциальности и коммуникации
- Центр поддержки и обратной связи

**3. Вкладка "Отчеты"**
- Аналитика потребления за выбранный период (день, неделя, месяц)
- Графики макронутриентов (белки, жиры, углеводы)
- Таблицы соответствия целевых и фактических показателей
- Детальный анализ распределения калорий по приемам пищи
- Отслеживание отдельных нутриентов

**4. Вкладка "Главная"**
- Лента новостей пользователей и друзей
- Социальные обновления о достижениях других пользователей
- Рецепты и фотографии еды из сообщества

## Пользовательский профиль и инициализация

### Процесс регистрации и настройки

**Этап 1: Определение цели**
При первом запуске пользователю предлагается выбрать одну из трех целей:
- Похудение
- Набор веса
- Поддержание текущего веса

**Этап 2: Сбор основных данных**
Приложение собирает следующую информацию:
- Пол
- Возраст (дата рождения)
- Рост (в см)
- Текущий вес (в кг)
- Целевой вес
- Уровень физической активности (низкий/умеренный/высокий/очень высокий)
- Количество килограммов для похудения/набора в неделю

**Этап 3: Регистрация аккаунта**
- Электронная почта
- Пароль
- Альтернативные методы: вход через Facebook или Google

**Этап 4: Автоматический расчет рекомендаций**
После регистрации приложение рассчитывает:
- РСК (рекомендуемое суточное количество калорий)
- Соотношение макронутриентов (белки, жиры, углеводы)

### Формулы расчета калорий

**Формула Миффлина-Сан Жеора (BMR - базовый метаболизм):**

Для мужчин:
\(BMR = (10 \times \text{вес в кг}) + (6.25 \times \text{рост в см}) - (5 \times \text{возраст в годах}) + 5\)

Для женщин:
\(BMR = (10 \times \text{вес в кг}) + (6.25 \times \text{рост в см}) - (5 \times \text{возраст в годах}) - 161\)

**Формула расчета TDEE (полная суточная потребность в калориях):**
\(TDEE = BMR \times \text{коэффициент активности}\)

Коэффициенты активности:
- Сидячий образ жизни: 1.2
- Слегка активный (тренировки 1-3 дня в неделю): 1.375
- Умеренно активный (тренировки 3-5 дней в неделю): 1.55
- Очень активный (тренировки 6-7 дней в неделю): 1.725
- Спортсмены (интенсивные тренировки): 1.9

**Корректировка для целей:**
- Похудение: TDEE - 20% (для более агрессивного) или - 10-15% (для умеренного)
- Набор веса: TDEE + 10-20%
- Поддержание: TDEE без изменений

### Управление целями и макронутриентами

**Установка пользовательских целей:**
- Возможность переопределения автоматических расчетов
- Установка целевых значений макронутриентов в граммах или процентах
- Сумма процентов должна равняться 100%
- Установка целевых значений для отдельных нутриентов

**Гибкое планирование по дням:**
- Разные дневные цели для разных дней недели
- Например: понедельник 2000 ккал, среда 2200 ккал, суббота 1800 ккал

## Система подсчета калорий и БЖУ

### Основные макронутриенты (БЖУ)

**Белки (Б)**
- Энергетическая ценность: 4 ккал на 1 грамм
- Строительный материал для мышц
- Производит чувство насыщения

**Жиры (Ж)**
- Энергетическая ценность: 9 ккал на 1 грамм
- Необходимы для усвоения жирорастворимых витаминов
- Поддерживают гормональный баланс

**Углеводы (У)**
- Энергетическая ценность: 4 ккал на 1 грамм
- Основной источник энергии
- Разделяются на простые и сложные

### Отслеживание микронутриентов

Приложение отслеживает и показывает:
- Пищевые волокна (клетчатка)
- Сахар (включая добавленный сахар)
- Натрий
- Холестерин
- Насыщенные жиры
- Витамины и минералы (в зависимости от версии)

### Методология расчета

**Система преобразования:**
- Один прием пищи = сумма всех добавленных в него продуктов
- День = сумма всех приемов пищи
- Калории высчитываются по формуле: (Белки × 4) + (Жиры × 9) + (Углеводы × 4)

## Управление пищевой базой данных

### Размер и качество базы

**Основные характеристики:**
- Более 1,9 миллиона проверенных вручную продуктов
- Все записи в базе проверяются на 100% для точности питательных веществ
- Включает продукты локальных брендов для разных регионов
- Постоянно обновляется новыми продуктами

### Категории продуктов в базе

1. **Продукты общих брендов**
   - Наиболее распространенные товары
   - Стандартные варианты приготовления

2. **Фирменные продукты**
   - Товары от известных производителей (Coca-Cola, McDonald's и т.д.)
   - Блюда ресторанных цепей

3. **Локальные и региональные продукты**
   - Специфические для каждого региона/страны
   - Традиционные блюда разных кухонь

4. **Рестораны и сети общепита**
   - Меню популярных ресторанов
   - Готовые блюда с указанной калорийностью

### Система поиска продуктов

**Методы поиска:**

1. **Текстовый поиск**
   - Поиск по названию продукта
   - Автодополнение при вводе
   - Быстрый доступ к недавно добавленным и часто используемым продуктам

2. **Поиск по штрихкоду (Barcode Scanner)**
   - Использование камеры смартфона
   - Автоматическое сканирование и определение продукта
   - Мгновенное добавление в дневник с указанием веса

3. **Поиск по изображению (AI-распознавание)**
   - Фотографирование продукта в фотоальбоме
   - Система пытается определить продукт по фотографии
   - Высокая точность для стандартных упакованных товаров

4. **Поиск по производителю/сети**
   - Возможность выбрать конкретного производителя
   - Найти товары конкретной торговой сети

### Добавление собственных продуктов

Процесс для продуктов, отсутствующих в базе:

**Шаг 1: Инициирование добавления**
- Поиск продукта в основной базе
- Если не найден, переход к добавлению нового

**Шаг 2: Заполнение данных о продукте**
- Название (обязательное)
- Торговая марка (опционально)
- Размер стандартной порции

**Шаг 3: Ввод пищевой информации**
- Калории (ккал)
- Белки (граммы)
- Жиры (граммы)
- Углеводы (граммы)
- Опционально: волокна, сахар, натрий и т.д.

**Источники информации:**
- Этикетка упаковки
- Официальные сайты производителей
- Справочники питательной ценности

## Функции дневника питания

### Структура ввода пищи

**Стандартные приемы пищи:**
- Завтрак
- Обед
- Ужин
- Перекус (или "Другое")

**Premium функция:**
- Расширение до 10 категорий приемов пищи для более детального отслеживания

### Методы добавления продуктов

**1. Поиск по названию (Вкладка "Еда")**
- Ввод названия продукта в строку поиска
- Выбор из списка результатов
- Указание размера порции в граммах или миллилитрах
- Автоматический расчет калорий и БЖУ для указанного размера

**2. Использование штрихкода**
- Нажатие на иконку камеры в интерфейсе добавления еды
- Выбор режима штрихкода
- Наведение камеры на штрих-код (красная линия должна пересекать код)
- Автоматическое определение продукта
- Проверка данных (внимание: возможны расхождения!)
- Указание весового размера порции

**3. Добавление через фотоальбом**
- Переход в фотоальбом (вкладка "Я")
- Нажатие на иконку камеры
- Выбор приема пищи
- Фотографирование продукта
- Система пытается определить продукт из фотографии
- Уточнение выбора из предложенных вариантов

**4. Ручное добавление новой еды**
- Поиск несуществующего продукта
- Нажатие кнопки "Добавить новую еду"
- Заполнение всех полей питательной информации
- Сохранение в личную базу

### Система сохраненной еды

**Сохраненные блюда (для регулярных комбинаций)**
- Сохранение часто едимых комбинаций продуктов
- Быстрое добавление в дневник одной кликом
- Вкладка "Сохраненная еда" в интерфейсе выбора продуктов

**Рецепты (для сложных блюд)**
- Описание полного рецепта с ингредиентами
- Указание количества порций
- Время подготовки и приготовления
- Пошаговые инструкции приготовления
- Вкладка "Рецепты" для доступа
- Возможность опубликовать рецепт в сообществе

### Создание собственного рецепта

**Структура рецепта:**

1. **Основная информация**
   - Название блюда
   - Краткое описание (видно другим пользователям)
   - Категория (завтрак, обед, ужин, другое)
   - Теги для поиска (опционально)

2. **Параметры приготовления**
   - Количество получаемых порций
   - Время подготовки (в минутах)
   - Время приготовления (в минутах)

3. **Ингредиенты (минимум 3)**
   - Название продукта
   - Вес/объем в граммах или миллилитрах
   - Вариант приготовления (сырой, вареный, запеченный и т.д.)
   - Автоматический расчет БЖУ для каждого ингредиента

4. **Инструкции приготовления**
   - Минимум 3 шага
   - Описание каждого шага
   - Возможность добавления большего количества шагов

5. **Автоматические расчеты**
   - Система вычисляет общие БЖУ всех ингредиентов
   - Делит полученные значения на указанное количество порций
   - Показывает БЖУ и калории на одну порцию

### Работа с порциями

**Указание размера порции:**
- В граммах (наиболее точно)
- В миллилитрах (для жидких продуктов)
- В стандартных единицах (чашка, стакан, штука и т.д.)

**Изменение размера порции:**
- Нажатие на добавленный продукт
- Корректировка веса в поле редактирования
- Автоматический пересчет калорий и БЖУ

### Опции и видимость дневника

**Подробное представление (Detailed View)**
- Активация из меню опций дневника
- Отображение верхней строки с суммой КБЖУ
- Видимость общих показателей калорий и нутриентов

**Синхронизация между устройствами**
- Данные синхронизируются через облако (Cloud Sync)
- Автоматическая синхронизация при наличии интернета
- Доступность данных на всех устройствах пользователя

## Отслеживание веса и прогресса

### Раздел "Мой вес"

**Добавление взвешиваний:**
- Дата взвешивания (автоматически текущая дата)
- Значение веса в килограммах
- Опционально: примечания к взвешиванию

**Доступ к функции:**
- Вкладка "Я" → "Мой вес"
- Нажатие иконки "+" для добавления нового значения

### График изменения веса

**Визуализация:**
- Линейный график в координатной системе (дата-вес)
- Отображение тренда похудения/набора веса
- Четкая визуализация прогресса со временем

**Анализ данных:**
- Расчет среднего снижения веса в неделю
- Определение траектории к целевому весу
- Прогнозирование даты достижения цели (при достаточных данных)

### Система достижений

**Типы достижений:**
- Потеря определенного количества килограммов (1 кг, 5 кг, 10 кг и т.д.)
- Регулярное ведение дневника (7 дней подряд, месяц и т.д.)
- Достижение целевого веса
- Логирование упражнений

**Мотивационная функция:**
- Отображение достижений в профиле
- Возможность делиться достижениями в сообществе
- Система "серий" (streaks) для постоянного использования

### Вкладка "Отчеты" - детальная аналитика

**Типы доступных отчетов:**

1. **Дневные отчеты**
   - Калории за выбранный день
   - Макронутриенты за день
   - Сравнение с целевыми значениями
   - Оставшиеся калории

2. **Недельные отчеты**
   - Общие калории за неделю
   - Усредненные дневные калории
   - График недельного потребления
   - Достижение целей за неделю

3. **Месячные отчеты**
   - Общие калории за месяц
   - Средние дневные калории
   - Тренды потребления
   - Сравнение с целями за месяц

**Анализируемые данные:**
- Соотношение калорий по приемам пищи
- Распределение БЖУ (графики)
- Отслеживаемые микронутриенты
- Соответствие фактического и целевого потребления
- Визуализация в виде графиков, диаграмм и таблиц

## Интеграция с внешними сервисами

### Синхронизация устройств и приложений отслеживания активности

**Поддерживаемые сервисы:**

1. **Google Fit**
   - Синхронизация шагов и активности
   - Передача данных о физической активности
   - Двусторонняя синхронизация

2. **Fitbit**
   - Импорт данных об активности и калорийности упражнений
   - Синхронизация веса
   - Синхронизация шагов

3. **Samsung Health**
   - Передача данных о шагах
   - Синхронизация активности
   - Синхронизация веса

4. **Garmin Connect**
   - Импорт данных о упражнениях
   - Синхронизация энергии, потраченной на деятельность
   - Импорт шагов

5. **Google Health Connect**
   - Новая унифицированная платформа синхронизации
   - Поддержка множества приложений и устройств

### Механика синхронизации

**Процесс синхронизации:**
- Подключение аккаунта через OAuth авторизацию
- Предоставление разрешений на доступ к данным
- Автоматическая синхронизация в фоновом режиме
- Периодическое обновление данных (обычно каждые несколько минут)

**Автоматический расчет расходуемых калорий:**
- Импорт количества шагов
- Импорт типов упражнений и их интенсивности
- Автоматический расчет сожженных калорий на основе личных данных
- Добавление к дневному дефициту/профициту калорий

### Направления синхронизации

**FatSecret → Другие приложения:**
- Экспорт данных о весе
- Возможность получения ссылки на экспорт данных
- Отправка данных по электронной почте

**Другие приложения → FatSecret:**
- Импорт активности (через Health Sync или аналогичные приложения)
- Синхронизация веса из устройств (умные весы)
- Синхронизация с медицинскими приложениями (опционально)

### Экспорт данных

**Функция экспортирования:**
- Доступна в основном дневнике (кнопка "Экспортировать" внизу)
- Для iOS: сохранение в облаке или отправка по почте
- Для Android: вход на полный веб-сайт → Еда → Добавить элемент → копирование ссылки

**Формат экспорта:**
- Обычно CSV или таблица
- Включает дневные данные о калориях, БЖУ и продуктах
- Может быть загружен в Google Sheets или Excel

### Система друзей и подписок

- Просмотр прогресса других пользователей

## API и интеграция для разработчиков

### Основные характеристики API

**FatSecret Platform REST API:**
- Используется более чем 10,000 разработчиков в 50+ странах
- Более 500 миллионов API вызовов в месяц
- Самая большая глобальная пищевая и пищевая база данных
- Бесплатное использование с возможностью коммерческого использования

### Аутентификация

**Методы аутентификации:**

1. **OAuth 1.0 (Signed Requests)**
   - Используется для поиска продуктов и получения информации о них
   - Не требует пользовательских данных
   - Подходит для публичных запросов к базе данных

2. **OAuth 2.0 (3-Legged OAuth)**
   - Используется для доступа к личным данным пользователя
   - Требует авторизации пользователя
   - Подходит для получения профиля, веса, истории приема пищи

### Основные методы API

**1. Поиск продуктов (foods.search / food.search.v5)**

Параметры:
- `query` - поисковая строка (название продукта)
- `max_results` - максимальное количество результатов (по умолчанию 10)
- `page_number` - номер страницы результатов

Возвращаемые данные:
- `food_id` - уникальный идентификатор продукта
- `food_name` - название продукта
- `food_type` - тип (Brand/Generic)
- `brand_name` - производитель (если применимо)
- `food_description` - описание продукта
- `food_url` - ссылка на полное описание на FatSecret

**2. Получение информации о продукте (food.get / food.get.v5)**

Параметры:
- `food_id` - ID продукта (полученный из поиска)

Возвращаемые данные:
- Полная информация о продукте:
  - Энергетическая ценность (калории)
  - Белки (г)
  - Жиры (г)
  - Углеводы (г)
  - Пищевые волокна (г)
  - Сахар (г)
  - Натрий (мг)
  - Холестерин (мг)
  - Размер стандартной порции

**3. Методы управления профилем**
- Получение профиля пользователя
- Получение истории веса
- Получение истории приема пищи
- Создание и обновление записей

### Эндпоинты API

**URL-базированная интеграция:**
```
https://platform.fatsecret.com/rest/food/v5?
food_id=33691&
oauth_consumer_key=YOUR_KEY&
oauth_signature=SIGNATURE&
oauth_timestamp=TIMESTAMP&
oauth_nonce=NONCE&
oauth_signature_method=HMAC-SHA1&
oauth_version=1.0
```

**Method-базированная интеграция:**
```
POST https://platform.fatsecret.com/rest/server.api
Parameters: method=food.get.v5&food_id=33691
```

### Аутентификация запросов

**Процесс подписания запроса (OAuth 1.0):**

1. Сбор параметров OAuth:
   - `oauth_consumer_key` - ваш публичный ключ
   - `oauth_nonce` - случайная строка
   - `oauth_timestamp` - текущее время Unix
   - `oauth_signature_method` - HMAC-SHA1
   - `oauth_version` - 1.0

2. Создание Signature Base String:
   ```
   HTTP_METHOD & ENCODED_URL & ENCODED_PARAMETERS
   ```

3. Генерация подписи с использованием HMAC-SHA1:
   - Ключ: `consumer_secret&access_secret`
   - Сообщение: Signature Base String

4. Добавление подписи в параметры запроса
