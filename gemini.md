# Правила работы с проектом FoodFlow Bot

## Язык документации

- Все планы реализации (`implementation_plan.md`) и списки задач (`task.md`) должны быть написаны **только на русском языке**.
- Сообщения об обновлениях и отчёты — на русском языке.
- Код (комментарии, переменные) — на английском (стандарт проекта).
- Общение с владельцем проекта — на русском.

## Режим работы AI

- По умолчанию — **read-only**. Код не менять без явного разрешения владельца.
- Файлы не удалять и не переносить.
- Перед правкой любого документа — читать реальный код.
- Не выдумывать факты — только то, что подтверждено кодом.

## Правила документации

- `README.md` — главный файл. Всегда синхронизирован с реальным кодом.
- `plans/roadmap_v1.md` — задачи и прогресс. Статусы проверять по коду.
- `планы/stability_audit_claude_2026-03-12.md` — аудит P0/P1/P2 проблем.
- `SECURITY_REPORT.md` — актуальные уязвимости и статус.
- `API_DOCUMENTATION.md` — документация REST API (FastAPI на порту 8001).
- `SHOPPING_MODE_PLAN.md` — план и статус реализации Shopping Mode.
- 🌐 API и Веб-интерфейс (WebApp)
- **Backend (API)**: FastAPI. Работает на порту **8001**.
- **Frontend**: Vite + React + TypeScript. Билд в `dist/`, раздача через FastAPI (`StaticFiles`) или Nginx.
- **VK Mini App**: Реализовано через `@vkontakte/vk-bridge`. Авторизация по `vk_user_id` через `vkLogin` API.
- **Nginx CSP**: Обязательно разрешить `frame-ancestors` для доменов VK (`vk.com`, `vk.ru` и др.).
- **Auth**: JWT токены, `GLOBAL_PASSWORD` для защиты.
